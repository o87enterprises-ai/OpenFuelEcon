## Phase 1: BLOCKERS (Weeks 1–2) — Unlock AdSense Approval

> **Goal:** Transform the "static calculator" signal into "comprehensive tool with publisher identity, structured content, and consistent localization." These tasks gate revenue directly.

### B1 — JS-side Locale Refactor

**Priority:** BLOCKER | **Revenue Impact:** 2 | **AdSense Impact:** 3 | **Effort:** M | **Dependencies:** None

**Why:** The cosmetic English leakage in the live widget, tip strings, currency display, and eco advice undermines the "localized experience" signal. AdSense reviewers may land on `/en-gb/` and see `$6.01` in the widget while the label says `£1.45/L` — reads as broken.

**What to do:**

1. Inject `window.FUELECON_LOCALE` global in each variant HTML head (modify `build_i18n.py` to inject inline script before closing `</head>`)
2. Add `t(key, fallback)` lookup helper to main calculator script that reads from `window.FUELECON_STRINGS` (also injected by build)
3. Move the following into locale JSONs under new top-level `"js"` key:
   - `tipsArray` (8 tip strings)
   - Widget row labels: "Live MPG", "Last trip", "Fuel used", "Premium · live metrics"
   - Green-zone messages: "GREEN ZONE · peak efficiency", "Moderate — easy win available", "High consumption — check route & tires"
   - Eco advice fragments in `calculateAndDisplay()`: highway savings message, tire pressure warning, acceleration tip
   - Saved vehicles placeholder text
4. Replace hardcoded `$` in `updateLiveWidget()` with `meta.currency_symbol` (read from injected locale meta)
5. Regenerate all 4 locale variants via `npm run build:i18n`

**Files Touched:**
- `public/index.html` (root only — build script handles variants)
- `scripts/build_i18n.py` (add JS injection logic)
- `i18n/en-us.json` (add `"js": { ... }` block)
- `i18n/en-gb.json`, `en-ca.json`, `en-au.json` (add matching `"js"` keys with localized text)
- `public/en-gb/index.html`, etc. ( regenerated)

**LOC:** ~150 (mostly JSON content)

**Success Criteria:** All user-facing strings on the live widget, tips section, and eco advice display in the correct locale's language and currency symbol. No `$` leakage on GBP/CAD/AUD variants.
